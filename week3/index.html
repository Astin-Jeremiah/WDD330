  1. Reading Notes JavaScript Novice to Ninja: Chapter 5
    • Object - self contained set of related values and functions
    • Object Literal - object that is created directly in the language by wrapping all its properties and methods in curly braces
      • const spiderman = {};
      • const spiderman = newObject();
    • Accessing properties of objects = use the dot notation
      • superman.name
      • superman[`name`]
    • Calling an object = use dot or bracket notation followed by parentheses after the method name
      • superman.fly()
      • superman[`fly`]()
    • In Operator - used to check if properties of a method exist
      • `city` in superman;
    • hasOwnProperty() = also used to check if properties of methods exist
      • superman.hasOwnProperty(`city`);
    • Finding all the properties of an object = loop though all of an objects properties and methods using a for in loop
      • for(const key in superman) {
        console.log(key + ":" + superman[key]);
        }
    • Object.key() = returns an array of all the keys of an object that is provided as an argument
      • for(const key of Object.keys(superman)) {
        console.log(key)
        }
    • Object.values() = returns an array of all the objects values
      • for(const value of Object.values(superman)) {
        console.log(value);
        }
    • Object.entries() = returns an array of key-value pairs
      • for(const [key,value] of
        Object.entries(superman)) {
        console.log(`${key} : ${value}`);
        }
    • Adding new properties to a method can be done at any time in a program; properties do not always appear in the order they are entered
      • superman.city = 'Metropolis';
    • Changing properties of an object can be done any time using assignments
      • superman['real name'] = 'Kal-El';
    • Delete Operator = removes properties from an object
      • delete superman.fly
    • Nested Objects - when an object contains other objects; values in nested objects are accessed by reference each property name in order using either dot or bracket notation
      • jla.wonderWoman.realName
    • Object literals can be passed as parameters to other functions
    • This - refers to the object that is is within; used to gain access to the objects parameters
    • Naming Collisions = when the same variable or function name is used for different purposes by code sharing the same scope
    • Object Literal Pattern = creating an object literal that serves as the namespace and then adds in any values or properties of that object and functions as methods
      • myCanvas.square()
    • JSON = JavaScript Object Notation; lightweight data-storage format that is used for data serialization and configuration
      • Property names must be double quoted
      • Permitted values are double quoted strings, numbers, booleans, arrays, and objects
      • Functions are not permitted values
      • parse() = takes a sting of data in JSON and format and returns a JavaScript object
      • stringfly() = takes a JavaScript object and returns a string of JSON data
    • Math Object - built in object that has several properties representing mathematical constraints
      • Math.PI = displays pi
      • Math.SQRT@ = the square root of 2
      • Math.SQRT1_2 = the reciprocal of the square root of 2
      • Math.E = Euler's constant
      • Math.LN2 = the natural logarithm of 2
      • Math.LN10 = the natural logarithm of 10
      • Math.LOG2E = the log base 2 of Euler's constant
    • Math.abs() = Absolute value; if number is positive it will remain the same, if number is negative it will become positive
    • Rounding Methods
      • Math.celi() = rounds a number up to the next integer or remains the same if already an integer
      • Math.floor() = rounds a number down to the next integer or remains the same if already an integer
      • Math.round() = round a number to the nearest integer
      • Math.trunc() = returns the integer part of the number only (chops off numbers after the decimal
    • Powers and Square Roots
      • Math.exp() = reaises a number to the power or Euler's constant
      • Math.pew(x,y) = raises any number (first argument) to the power of another number (second argument)
      • Math.sqrt() = returns positive square root of a number
      • Math.cbrt() = returns the cube root of a number
      • Math.hypot() = returns the square root of the sum of the squares of all its arguments (calculate hypothenuse of a right-angled triangle)
    • Math.log() = returns the natural logarithm of a number
    • Maximum and Minimum Methods
      • Math.max() = returns the maximum number from its arguments
      • Math.min() = returns the minimum number from the given arguments
    • Trigonometric Functions
      • Math.sin() = returns the sine of an angle
      • Math.con() = returns the cosine of an angle
      • Math.tan() = returns the tangent of an angle
      • Math.asin() = returns the arcsine of a number; result is an angle
      • Math.acos() = returns the arccosine of a number; result is an angle
      • Math.atan() = returns the arctangent of a number; result is an angle
    • Random Numbers
      • Math.random() = creates a random number; number will be between 0 and 1
      • Multiple Random numbers function to increase random numbers; 6*Math.random() = random numbers between 0 and 6 with decimal values
      • For random whole numbers use Math.floor(); Math.floor(6*Math.random())= random whole numbers between 0 and 6
    • Date Objects
      • Create New Date Object
        • const today = new Date();
        • today.toString() = 'Tue Feb 14 2017 16:35:18 GMT+000 (GMT)'
      • If a specific date is not specified the default is the current date and time
        • const christmas = newDate('2017 12 25')
      • Getter Methods
        • getDay() or getUTCDay() = used to find the day of the week that the date object falls on; returns a number between 0 and 6; 0=Sunday...6=Saturday
        • getDate() or getUTCDate() = returns the day of the month for the date object; returns actual number of the date
        • getMonth() or get UTCMonth() = find the month of the date object; returns a number 0 to 11; 0=January....11=December
        • getFullYear() or getUTCFullYear() = returns the year of the date object
        • getTime() = returns a time stamp representing the number of milliseconds since the Epoch
          • A day = 1000*60*60*24 milliseconds
        • getTimezoneOffset() = returns the difference in minutes between the local time on the computer and UTC
      • Setter Mehods
        • setDate() = sets day of event; returns timestamp of updated date object
        • setMonth() = sets month of event; returns timestamp
        • setFullYear() = sets year of event; returns timestamp
        • setTime() = sets time stamp for event; must use millisecond time stamp
    • Regular Expressions - Pattern that can be used to search strings for matches to a pattern; common with find and replace types of operations
      • Creating Regular Expressions
        • Using literal notation of writing the regular expression between forward slashes
          • const pattern = /[a-zA-Z+ing$/;
        • Creating a new instance of the regular expresion using a new operator and constructor
          • const pattern = newRegExp('[a-zA-Z]+ing');
      • Regular Expression Methods
        • test() = see if a string passed to the methods as a parameter matches the regular expression; returns true/false answer
          • pattern.test('joke');
        • exec() = works same way as test but return's an array containing the first match found or null if no matches are found
          • pattern.exec('joke');
      • Character Groups - groups of characters placed together inside square brackets representing any one of the characters inside the brackets
        • const vowels = /[aeiou];/
        • const alphabet = /[A-Z]/;
        • const numbers = /[0-9]/;
        • const notcapital = /[^A-Z]/; ^ represents is not
      • Regular Expression Properties
        • Global - Makes pattern return all matches; default only looks at first occurrence
          • pattern = /java/g
        • Ignore Case - Makes the pattern case insensitive; by default they are case sensitive
          • pattern = /java/i
        • Multiline - Makes the pattern multiline; default will stop after the end of the line
          • pattern = /java/m
      • Special Characters/Metacharacters
        • . = matches any character; except line breaks
        • \w = matches any word character; equivalent to [A-Za-z0-9]
        • \W = matches any non-word character; equivalent to [\^A-Za-z0-9_]
        • \d = matches any digit character; equivalent to [0-9]
        • \D = matches any non digit character; equivalent to [^0-9]
        • \s = matches any white space character; equivalent to [\t\r\n\f]
        • \S = matches any non-white space character; equivalent to [^\t\r\n\f]
      • Modifiers - placed after a token to deal with multiple occurrences of that token
        • ? = makes the preceding token in the regular expression optical
        • * = makes one or more occurrences of the preceding token
        • + = matches one or more occurrences of the preceding token
        • {n} = matches n occurrences of the preceding token
        • {n,} = matches at least n occurrences of the pattern
        • {,m) = matches at most n occurrences of the preceding token
        • {n,m} = matches at least n and at most m occurrences of the preceding token
        • ^ = marks the position immediately before the first character in the string
        • $ = marks the position immediately after the first character in the string
      • Regular Expressions and Strings
        • split() = can be used to split a string into the separate elements of an array
        • match() = used to return an array of all matches
        • search() = return the position of the first match
        • replace() = replace any matches with another string
  2. Reading Notes JavaScript Novice to Ninja: Chapter 6
    • Document Object Model (DOM) = allows you to access elements of a web page and enable interactions with the page by adding and removing elements, changing the order, content and attributes of elements, and even altering how they are styled
    • Node = special object with a number of properties and methods that we can use to find information about or modify
    • node.nodeType = displays the type of node; identified by a number
      • 1 = element
      • 2 = attribute
      • 3 = text
      • 4 = comment
      • 9 = body
    • node.nodeName = find the name of the element
    • Document.body = returns the body element of a website
    • Document.images = returns a node list of all the images contained in the document
    • Document.links = returns a node list of all the <a> and <area> elements that have an href reference
    • Document.anchors = returns a node list off all the <a> elements that have a name attribute
    • Document.forms = returns a node list off all the forms on a document
    • Array.ffrom() and the spread operator [...] can be used to convert a node list into an array
    • getElementById() = returns a reference to the element with an unique id attribute that is given as an argument; if no element exists the ID provided null is returned
    • Every ID attribute should be unique to just one element
    • getElementsByTagName() = return a live node list of all the elements with the given tag name; example: get all list items in a document; if no elements in the document with the given tag name an empty node list is returned
    • getElementsByClassName() = return a live node list of all elements that have the class name this is specified as an arguments
    • document.quearySelector() = allows you to use CSS notation to find the fist element in the document that matches a CSS selector provided as an argument if no element match it will return an empty node list
    • document.quearySelectorAll() = use CSS notation but returns a node list of all the elements in the document that match the CSS query selector; if no elements match it will return an empty node list
    • childNodes = lists all the nodes that are children of the node concerned
    • firstChild = returns the first child of a node
    • lastChild = returns the last child of a node
    • parentNode = returns the parent node of an element
    • nextSibling = returns the next adjacent node of the same parent
    • previousSibling = return the previous adjacent node
    • nodeValue = returns the text contained inside a referenced text node
    • textContent = returns the text content of an element as a string
    • getAttribute() = returns the value of the attribute provided as an argument; if an element does not have the given attribute null is returned
    • setAttribute() = change the value of an elements attribute; takes two arguments the attribute you wish to change and the value of the new attribute
      • wonderWoman.setAttribute('class', 'villian');
    • className = allows the class of an element to be set directly; can also be used to find out the value of the class attribute
    • classList = lists all the classes an element has
      • classList.add() = adds a class to an element without overwriting any classes that already exist
      • classList.remove() = remove a specific class from an element
      • classList.toggle() = add a class if an element doesn't have it already and remove the class if it does already
      • classList.contains() = checks to see if an element has a particular class
    • createElement() = takes a tag name as a parameter and returns that element
    • document.createTextNode() = creates a text node from a string parameter
    • appendChild() = adds another node given as an argument as a child node
    • Function to create HTML elements
      • Function createElement(tag, text) {
        const el = document.createElement(tag);
        el.textContent = text;
        return el;
        }
    • insertBefore() = place a new element before another element in the markup; called on the parent node; takes two parameters the new node to be added and the node you want it to go before
      • heroes.insertBefore(aquaman,wonderWoman);
    • removeChild() = removes an element from the page
    • replaceChild() = replace one node with another; called on the parent node; takes two parameters the new node and the node to be replaced
      • h1.replaceChild(newText,oldText);
    • innerHTML = returns all the child elements of an element as string of HTML
    • Using style properties to update CSS; CSS properties with dashes are converted to camel case; elements can be hidden using display hide and display block
      • superman.style.border = "red 2px solid";
    • getComputedStyle() = retreive all the style information of an element given a parameter; read only to find out information
  3. Reading Notes JavaScript Novice to Ninja: Chapter 7
    • Event Listener = checks to see if a particular action is done in order to activate the JavaScript code
      • Document.body.addEventListener("click", doSomething);
    • Inline Event Handling = the event listening goes into the HTML code; not best practice
    • Using Event Handlers = uses the event handler properties that all node object have; restricted because you can only have function or each event
      • document.onclick = function(){
        console.log('You clicked on the page!');}
    • Using Event Listeners = allows multiple functions to be attached independently to different events
      • Document.body.addEventListener('click, doSomething)
      • addEventListener('click', () => alert(;You Clicked!));
      • function doSomething() {
        alert('You Clicked!');
        }
        addEventListener('click,doSomething);
    • event.type = logs the type of event such as a click
      • console.log(event.type);
    • event.target = logs the type of event with reference to the mode that fired the event
      • console.log(event.target);
    • Finding The Position Where The Mouse Event Occurred
      • Screen X, Screen Y = show the number of pixels from the left and top of the screen respectively where the end took place
      • Client X, Client Y = show the number of pixels from the left and top of the client that is being used
      • Page X, Page Y = shows the number of pixels from the left and top respectively where the event took place on the document; takes into account the page being scrolled before the event
    • Mouse Events
      • click = when the mouse is clicked
      • mousedown & mouseup = occur before a click happens
      • dblclick = the mouse was double clicked; you should not place a click and double click event on the same element
      • mouseover = when the mouse pointer is placed over the element
      • mouseout = when the mouse pointer moves away from an element
      • mousemove = whenever the mouse moves
    • Keyboard Events
      • keydown = when a key is pressed; continues to occur if the key is held down
      • keypress = occurs after a keydown but before a keyup event; only works on keys that produce input
      • keyup = when a key is released
      • Modifer Keys = Shit, Ctrl, Alt, meta (windows)
    • Touch Events = used to record events when using mobile devices with a touch screen; you should support both touch and non-touch events because you do not know how the user will be using the application; click sill fires on touch screens but there is a small delay
      • touchend = user stops touching the surface
      • touchmove = user has touched the screen then moves around without leaving; occurs as long as the user is touching the screen
      • touchenter = a user has already started touching the surface but passed over the element which the event listener is attached
      • touchleave = the user is still touching the surface but leave the element to which the event listener is attached
      • touchcancel = a touch event is interrupted; user finger moves outside the document window or multiple fingers being used at once
      • touches = list of touch objects that represent all the touches taking place on the device; length property that tells you how many touch points are in contact with the screen
    • removeEventListener() = removes an event listener
    • preventDefault() = stops a default behavior from happening; use carefully users expect certain behaviors
    • Bubbling = when the event fires on the element clicked on first, then bubbles up the document tree firing an event on each parent element until it reaches the root node; bubbling is a default behavior
      • event.stoprPropagation() = stops bubbling from happening
    • Capturing = firing an element on the root element, which propagates downward firing of an event on each child element until it reaches the target element that was clicked on
