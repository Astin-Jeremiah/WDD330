Using Fetch Notes

  • Fetch API provides a JavaScript interface for accessing and manipulating parts of the HTTP pipeline and provides a global fetch() method that provides an easy logical way to fetch resources asynchronously across the network
  • Fetch differs from AJAX in three main ways
    • The promise returned from fetch() won't reject on HTTP error status and will only reject on network failure or if anything prevented the request from completing
    • fetch() can receive cross-site cookies
    • fetch() wont send cookies unless you set it in the credentials
  • The Headers interface allows you to create your own headers for you site
    • Guard allows you to mute your header from outside site
      • none: default
      • required: guard for a header object obtained from a request request.headers
      • required-no-cors: guard for a header object obtained from a request created with request.node
      • response: guard for a header obtained from a response response.header
      • inmutable: mostly used for service workers, renders the header object read only
  • Response - what is returned when a fetch() promise is resolved
    • Response.status - an integer containing the response status code
    • Response.statusText - a string which corresponds to the HTTP status code message
    • Response.ok - shorthand for checking that the status is in the range of 200-299
