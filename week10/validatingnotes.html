Validating Forms Notes

  • Client Side Form Validation - ensure all required form controls are filled out in the correct format
  • Client Side Form Validation should not be considered an exhaustive security measure. Your program should always perform security checks on any form submitted data on the server side as well as the client side.
  • Built In Form Validation - uses HTML5 form validation features to validate the form
    • required: specifies whether a form field need to be filled in before the form can be submitted
    • minlenght and maxlenght: specifies the minimum or maximum length of textual data for the entry
    • min and max: specifies the minimum and maximum values of numerical input types
    • type: specifies whether the data needs to be a number, an email address, or some other specific preset type
    • pattern: specifies a regular expression that defines a pattern the entered data needs to follow; regular patterns are case sensitive
      • a - matches one character that is a (not b or aa, or so on)
      • abc - matches a, followed by b, followed by c
      • ab?c - matches a, optionally followed by c, followed by c (ac or abc)
      • ab*c - matches a, optionally followed by b's, followed by c (ac, abc, abbbbbc, and so on)
      • a|b - matches one character that is a or b
      • abc|xyz - matches exactly abd or exactly xyz (not abcxyz or a or y or so on)
  • JavaScript Form Validation - Form validation is custom coded via JavaScript
    • Most browsers support the Constraint Validation API
      • validateMessage - returns a localized message describing the validation constraints that the control doesn't satisfy, if no constraints are violated it will return an empty string
      • validity - returns a ValidityStatment object that contains several properties describing the validity state of the elements
        • patternMismatch - returns true if the value does not match the specified patter and false if it does match
        • tooLong - returns true if the value is longer than the maximum length specified by the maxLength attribute and false if it is shorter or equal to the maximum
        • tooShort - returns true if the value is shorter than the minimum length specified by the minlength attribute or false if it is greater or equal to the minimum
        • rangeOverflow - returns true if the value is greater than the maximum specified by the max attribute or false if it is less than or equal to the maximum
        • rangeUnderflow - returns true if the value is less than the minimum specified by the min attribute or false if it is greater than or equal to the minimum
        • typeMismatch - returns true if the value is not in the required syntax (email or url) or false if the syntax is correct
        • valid - returns true if the element matches all its validation constraints and is therefor considered to be valid or false if it fails any constraints
        • valueMissing - returns true if the element has a required attribute but not value or false otherwise
        • willValidate - returns true if the element will be validated when the form is submitted and false otherwise
        • checkValidity() - returns true if the elements value has no validity problems and false otherwise
        • setCustomValidity(message) - adds a custom message to the element, the message is displayed if the element is considered invalid
    • You can write your own validation code, but you will want to consider questions.
      • What kind of validation should I perform?
      • What should I do if the form doesn't match?
      • How can I help the user to correct invalid data?
