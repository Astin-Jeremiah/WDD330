  1. Reading Notes JavaScript Novice to Ninja: Chapter 2
    • JavaScript single line comment //
    • JavaScript multi line comment /*...*/
    • It is best practice to write each JavaScript statement on a new line and terminate each line with a semicolon.
    • A block is a series of JavaScript statements that are collected together using curly braces; blocks do not need to be terminated with a semicolon.
    • Variables - a value stored in the program's memory; must be declared before they can be used
      • Const - A constant variable not assigned to another value
      • Let - if the variable might be reassigned later in the program
    • You can assign multiple variables on the same line of code as long as you separate them by commas.
    • Global Scope - Variables are declared outside the block; variable is accessible anywhere in the program; should be kept to a minimum
    • Local Scope - Variable is only defined within the block; variables cannot be accessed outside the block
    • Variable names can start with upper or lower case letters, an underscore or dollar sign; variables can contain number but they cannot start with one; variable names are case sensitive
    • Camel Case - starts with a lower case letter and then every new word is capitalized firstNameAndLastName
    • Underscore - separates each new word with an underscore firs_name_and_last_name
    • Strings - collection of characters such as letters and symbols
      • Written in single quotation marks 'Hello World'
      • If the string contains an apostrophe use double quotes "It's A Small World After All"
    • Symbols - create unique values, do not have a literal form
      • Use the symbol function: const uniqueID = Symbol('this is a unique ID');
    • Numbers - can be integers (whole numbers), floating points (decimals), or exponential numbers (1e6)
    • Infinity - numbers too big or small for JavaScript to deal with
    • NaN - Not A Number; a numerical operation is attempted but the result is not numerical
    • Undefined - variables that have not been assigned to a value
    • Null - no vales; placeholders
    • Booleans - True/False values
    • Logical Operations
      • Logical Not - Returns the opposite of a boolean place ! in front of the value
      • Logical And - Only evaluates to true if both values are true; use the && symbol
      • Logical Or - Evaluates to true if any of the operators are true; use the || symbol
    • Equality
      • Two Values Are Equal To Each Other ==
      • Two Values Are Not Equal To Each Other !==
      • Two Values Are Greater Than >
      • Two Values Are Less Than >
      • Two Values Are Greater Than Or Equal To >=
      • Two Values Are Less Than Or Equal To <=
  2. Reading Notes JavaScript Novice to Ninja: Chapter 3
    • Arrays - Ordered lists of values
    • Manipulating Arrays
      • .pop() – removes the last item from an array
      • .shift() – removes the first item from an array
      • .push() – appends new value at the end of the array
      • .unshift() – appends new value at the start of the array
      • .join() – turns the array into a list of stings separated by commas ( the separator can be changed by placing it in parentheses .join(‘&’)
      • .slice() – creates a subarray by chopping out a slice of the original array; used to delete data from array as well as append in specific places
      • .reverse() – permanently reverses order of the array
      • .sort() – permanently arranges the array alphabetically
      • .indexOf() – find the first occurrence of a value in an array; if not in the array displays -1
      • .includes() – a Boolean version of indexOf()
    • Sets - Data structure that represents a collection of unique values; cannot have any duplicate values
    • Manipulating Sets
      • Creating Sets const list = new Set();
      • .add() - adds values to a set
      • .size() - determines the number of values in a set
      • .delete() - removes values from a set
      • .clear() - removes all values from a set
      • [...] - Spread Operator; converts the set into an array literal
    • Maps - Data structure that keeps a list of key and value pairs
    • Manipulating Maps
      • Creating Maps const romanNumerals = new Map();
      • .set(1, 'I') - add a key and value pair to a map
      • .get() - looks up a value in a map
      • .has() - check if a particular key is in a pay; returns a boolean value
      • .delete() - remove a key and value pair from a map
      • .clear() - remove all key and value pairs from a map
      • Array.from() or spread operator [...] can be used to convert a map into a nest array of key-value pairs
    • If Statement - The code inside the block will only run if the conditions in parentheses is true
      • if (condition) {
        code to run if true
        }
    • Else Statement -Add an alternative block of code to run if the condition is false
      • if (condition) {
        code to run if true
        } else {
        code to run if condition is false
        }
      • Ternary Operator - Shorthand way to write an else statement
        • condition ? (//code to run if condition is true) : (//code to run if condition is false)
    • Switch Operator - Used When your code has lots of conditions to test
      • switch (number) {
        case4:
        console.log('You rolled a four');
        break;
        case 5:
        console.log('You rolled a five');
        break;
        case 6:
        console.log('You rolled a six');
        break;
        default:
        console.log('You rolled a number less than four');
        break;
        }
    • Loops - Repeat a piece of code over and over again according to certain conditions; if the conditions are not met the loop will get stuck in an infinite loop until it crashes the program
    • While Loop - repeatedly runs a block of code while a certain condition is true
      • while (condition) {
        do something
        }
    • Do...While Loop - similar to a while loop but the conditions come after the block of code; loop always runs once regardless of the condition being true or not
      • do {
        do something
        } while (condition)
    • For Loop - has three parts initialization code that runs before the loop starts, a condition that has to be satisfied for the loop to continue, and after code that is what to date after each loop iteration
      • for (initialization ; condition ; after {
        do something }
      • For Loops can be nested inside another loop so that the inner loop will run all the way though before the next step of the outer loop occurs
      • For loops can be used to loop over arrays, sets and maps
  3. Reading Notes JavaScript Novice to Ninja: Chapter 3
    • Functions - a chunk of code that is referenced by a name and is almost like a small, self contained mini program
    • Function Declaration
      • function hello(){
        console.log('Hello World!');
        }
    • Function Expression
      • const goodbye = function bye(){
        console.log('Goodbye World!');
        };
    • Function() Constructors
      • const hi = new Function('console.log("Hi
        World!");');
    • Invoking A Function - run the code inside the function's body; simply enter the functions name followed by parentheses
    • DRY - DONT REPEAT YOURSELF - code should not only be written once and not duplicated multiple places in a program
    • Return Values- what the function returns after completing the code; comes after the keyword return
    • Parameters and Arguments - the values provided to the function for input
    • Arrow Functions - abbreviated syntax for declaring functions; parameters come before the arrow and the main body of the function come after
      • const square = x => x*x;
      • Single parameters do not need to be put in parentheses
      • Body of the function doesn't need placed inside a block if it's only one line
      • Return keyword is not required if the return statement is the only statement in the body of the function
      • If the function doesn't require any parameters a pair of empty parentheses must go before the arrow
      • Longer functions will still require curly braces to delimite the body of the function and the return keywords at the end
      • Best for short anonymous functions
    • Callbacks - A function that passes parameters to another function
    • .map() - iterates over an array and takes a callback function as a parameter that is invoked on each item in the array
      • [1,2,3].map(square);
        [1, 4, 9]
    • .reduce() - iterates over each value in an array and cumulatively combines each result to return just a single value
      • [1,2,3, 4, 5].reduce( (acc,val) => prev + val);
        15
    • .filter() - returns a new array that only contains items from the original array that return true when passed to the callback
      • const numbers = [2, 7, 6, 5, 11, 23, 12]

        numbers.filter(x => x%2 === 0); returns true if the number is even

        [2, 6, 12]
