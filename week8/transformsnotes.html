CSS3 Transforms and Transitions

  • Transform - lets you translate, rotate, scale, and or skew any element on the page; do not work on inline elements; use display:inline-block to over come this
  • Translated elements can only be moved relative to its current position
  • translate(x,y) - moves an element x from the left and y from the top
  • translatex and translatey can be used to only move the object in one direction
  • scale(x,y) - scales an element by the defined factors horizontally and vertically; provide only one value to scale the same amount in both directions preserving the original aspect ratio
  • scalex and scaley can be used to only scale the element in one direction
  • Use a space separated list when declaring multiple transform functions to the same element
  • rotate() - rotates an element around the point of origin by a specified angle value; default point or origin is the objects center; angels are declared in degrees with positive and negative numbers to move clockwise or counterclockwise; values can also be in grads, radians, or turns
  • skew(x,y) - specifies a skew along the x and y axis; if only one number is proved it will only skew on the x axis
  • skewx and skewy can be used to skew in only one direction
  • transform-origin - CSS property to control the origin from which the transforms are applied center of object is the default
  • Creating A Simple Transition Using Only CSS
    • Declare the original state of the element in the default style declaration
    • Declare the final state of your transitioned element: a:hover state
    • Include the transition functions in your default styling declaration using the transition properties: transition-property, transition-duration, transition-timing-function and transition-delay
  • transition-property - defines the CSS properties of the element that should be transitioned
  • transition-duration - sets how long the transition will take or the time it takes to go from the default sate to the transitioned sate; specified in seconds or milliseconds
  • transition-timing-function - lets you control the pace of the transition in even more granular detail; ease, linear, ease-in, ease-out, ease-in-out
  • The timing function be be more precise by defining a cubic-bezier function with four numeric parameters; advanced technique
  • transition-delay - introduce a delay before the transition begins; defined in number of seconds or milliseconds to delay the transition
  • Transition shorthand - combine multiples lines of transition properties to one single statement
  • The transition property allows for multiple transitions on one call; example change the color at the same time as size and rotation
  • Keyframe - snapshot that defines a staring and end point of any smooth transition
  • animation-name - attaches an animation to the element
  • animation-duration - defines the length of time (in seconds or milliseconds) an animation takes to complete one iterations
  • animation-timing-function - determines how the animation will progress over its duration (same options as transition-timing-function
  • animation-iteration-count - define how many times the animation will play through; usually expressed as a number or the value infinite; 1 is the default value
  • animation-direction - what direction the animation runs start to finish or finish to start; reverse or alternate are common values
  • animation-delay - define how many milliseconds or seconds to wait before the browser begins the animation
  • animation-fill-mode - defines what happens before the first animation iterations begins or after the last animation iteration concludes; none, forward, backwards, or both
  • animation-play-sate - defines whether the animation is running or paused
  • Animation Shorthand used to combine all the animations elements into a single CSS statement
