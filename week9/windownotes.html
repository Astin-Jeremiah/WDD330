The Window Object Notes

  • Global Object - all variables that are created in the global scope are actually properties of a global object and any functions are methods of it
  • Browser Object Model - collection of properties and methods that contain information about the browser and computer screen
  • window.alert() - pause the execution of the program and display a message in a dialog box
  • window.confirm() - stop the execution of the program and display a confirmation dialog that shows the message provided as an argument and give the options of OK or cancel; returns boolean value of true if the user presses OK and false if the user clicks cancel
  • window.prompt() - stop the execution of the program and display a dialog that shows a message provided as an argument as well as an input field that allows the user to enter text; the text is returned as a string when the user clicks OK, if the user clicks cancel null is returned
  • window.navigator.userAgent - returns information about the browser and operating system being used; info can be masked by the user
  • window.location - contains information about the URL of the current page
    • href - returns the full URL as a string
    • host - returns a string describing the domain of the current URL and the port number; port might be omitted if the default port 80 is used
    • hostname - returns a string describing the domain of the current URL
    • port - returns a string describing the port number; will remain empty if the port is not explicitly stated in the URL
    • pathname - returns a string of the path that follows the domain
    • search - returns a string that starts with a ? followed by the query string parameters; contains an empty string if there are no query parameters
    • hash - returns a string that starts with a # followed by the fragment identifier; returns an empty string is there is no fragment identifier
    • origin - returns a string that shows the protocol and domain where the current page originated from; property cannot be changed, read only
    • reload() = used to force a reload of the current page; if given the parameter of true it will force the browser to reload the page from the server instead of a cached page
    • assign() - used to load another resource from a URL provided as a parameter
    • replace() - similar to assign() nut the current page will not be stored in the session history; so the user is unable to navigate back using the back button
    • toString() - returns a string containing the whole URL
  • window.history - used to access information about any previously visited pages in the current browser session
  • window.history.length - shows how many pages have been visited before arriving at the current page
  • window.history.go() - used to go to a specific page where 0 is the current page
  • window.history.forward() and window.history.back() - used to navigate forward and backward by one page respectively; just like the forward and back buttons
  • window.open() - opens a new window
  • close() - close a window as long as you have a reference to the window
  • window.moveTo() - moves a window; takes two parameters an X and Y coordinate to move the window to
  • window.resizeTo() - resize a window; takes two parameters the width and height of the resized window
  • window.screen - object contains information about the screen the browser is displayed on; you can find out the height and width of the screen in pixels
  • availHeight and availWidth - find the height and width of the screen excluding the operating system menus
  • colorDepth - find the cold depth of the user's monitor
  • document.write() - writes a sting of text to the page; HTMl can be included in the string and becomes part of the DOM
  • Cookies - small files that are saved locally on a user's computer; stores information that can then be retrieved between page request; can be used to personalize a user's browsing experience, storing user preference, keeping track of user choices, authentication and tracking purposes; take the form a a text file that contain a list of name/value pairs separated by semicolons
  • Create a cookie by assigning it to a JavaScript cookie jar using the document.cookie property
  • Cookie values can be changed by reassigning to to document.cookie
  • You can view the current contents of a cookie with document.cookie
  • Use the split method to break a cookie string apart into an array containing each name/value pair and then use a loop to iterate through the array
  • By default cookies are session cookies and are deleted once a browser session is finished (user closes browser) this can be extended by adding ; expires=date to the end of the cookie when it is set; you can also set the max-age value which takes a value in seconds
  • By default cookies can only be read by pages inside the same directory and domain as the file was set; security reason so that access to the cookie is limited; paths can be changed so that any page on the root directly can read the cookie by adding ; path=/ to the end of the cookie set; you can also set 'domain=donainName at the end of the cookie
  • A cookie can be secured by adding 'secure to end of the cookie; can only be transmitted over a HTTPS network
  • To remove a cookie set it to expire at a time in the past; session cookies expire when the tab or browser is closed
  • window.setTimeout() - accepts a callback to a function as its first parameter and number of milliseconds or seconds as the second parameter; schedules the execution of a function
  • window.clearTimeout() - cancels a time out
  • window.setInterval() - works similar to setTimeout but will repeatedly invoke the callback function after every given number of milliseconds
  • window.clearInterval() - stops/clears a setInterval
  • setTimeout() and setInterval() can be used to animate elements on a page
