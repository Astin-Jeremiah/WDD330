HTML 5 API Notes

  • data- Attribute - way of embedding data in a web page using custom attributes that are ignored by the browser; their sole purpose is to assist a JavaScript program
    • start with data-
    • contain only lowercase letters, number, hyphens, dots, colons or underscores
    • include an optional string value
  • Web Storage API - provides key value stored on a clients computer that is similar to a cookie but has fewer restrictions, more storage capacity, and is generally easier to use; also know as Local Storage; only strings can be saved to local storage
    • Information stores is not shared with the server on any request
    • Information is available in multiple windows of the browser, but only if the domain is the same
    • Storage capacity is much larger than 4KB; most browsers have a limit of 5GBper domain
    • Any data stored does not automatically expire as it does with cookies
    • localStorage.setItem() - saves data to local storage
    • localStorage.name - can be used to change the name of the local storage element
    • localStorage.removeItem or delete localStorage.name - are used to remove items from local storage
    • localStorage.clear() - removes everything stored in local storage
    • Use JSON.stringify() and JSON.parse() to convert objects from JavaScript to local storage
  • Geolocation API - used to obtain the geographical position of the device; find the users exact location; due to privacy concerts permission to use this data must be grated by the user
    • getCurrentPosition() - returns a position object to a specified call back function; returns the longitude and latitude of the device
    • position.speed - returns the ground speed of the device in meters per second
    • position.altitude - returns an estimate of the devices altitude in meters above the center of the earth
    • position.heading - returns the direction the device is moving in; measured in degrees clockwise from North
    • position.timestamp - returns the time the position information was recorded
    • position.accuracy - returns the accuracy of the latitude and longitude properties in meters; the lower the returned value the more accurate the measurement
    • position.altitudeAccuracy - returns the accuracy of the altitude property in meters
    • watchPosition() - will call a callback function every time the position of the device is updated
    • clearWatch() - stops the watch callback from being called
  • Web Workers - allow processes to be run in the background, adding support for concurrency in JavaScript; prevents the script has become unresponsive error message if the process hakes a while to be carried out
    • Worker() - creates a new worker
    • Web browsers use the concept of messages to communicate back and forth between the main script and the worker script
    • postMessage() - used to send a message and start the worker working
    • terminate() or close() - used to stop a worker when they have completed its task
  • Service Workers - allow a worker script to run in the background with the added benefit of being able to intercept network requests; allowing you do take alternative action if the network is offline and effectively creates an app like offline experience; allow access to push notifications or background sync; require a secure network to run on HTTPS to avoid any malicious code hijacking network requests
  • Websockets - allows two way communication with a server; also known as push messaging; the connection is kept open and resources are pushed to the client as soon as they are received
  • Notifications API - allows you to show messages using the system's notifications; usually a pop up in the corner of the screen
    • requestPermission() - before you send notifications you need to get permission granted from the user
    • close() - will close the notification
  • Multimedia - HTML 5 introduced audio and video tags that can be used to insert audio and video clips into a webpage; a Media API was also included to control media playback using JavaScript
    • play() - starts the clip playing at its current position
    • pause() - pauses the clip at the current position
    • volume - number that can be used to set the audio volume
    • muted - boolean value that can be used to mute the audio
    • currentTime - number value that can be used to jump to another part of the clip
    • playbackRate - used to fast-forward or rewind the clip by changing the playback speed
    • loop - boolean value that can be set to true to make the clip repeat on a loop
    • duration - used to see how long the clip lasts
    • play event - fires when the clip starts and when it resumes after a pause
    • pause event - fires when the clip is paused
    • volumechange event - fires when the volume is changed
    • loadmetadata event - fires when all the video's metadata is loaded
  • A shim or polyfill is a piece of code that adds support of missing features to older browsers
