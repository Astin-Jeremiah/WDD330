Testing and Debugging Notes

  • System Error - there's a problem with the system or external devices with which the program is interacting
  • Programmer Error - the program contains incorrect syntax or faulty logic; sometimes as simple as a typo
  • User Error - the user has entered data incorrectly and the program is unable to handle it
  • Exception - an error that produces a return value that can then be used by the program to deal with the error; function is not defined
  • Stack Trace - a sequence of functions or method calls that lead to the point where the error occurred; works backwards from the point at which the error occurred to id the original function or method that started the sequence
  • Warning - occurs if there's an error in the code that isn't enough to cause the program to crash
  • Browsers vary in how they display warnings and exceptions; Some display a small icon in the order of the browser and others require you to open the console
  • When a runtime error occurs in the browser the HTML will still appear on the page but the JavaScript code will stop working; In the case of a warning the JavaScript will continue to run, but may run incorrect
  • Strict Mode
    • Introduced in ECMAScript 5
      • Produces more exceptions and warnings and prohibits the use of some deprecated features
    • Using strict mode can help improve your coding clarity and speed
    • 'use strict' is added to the first line of the JavaScript file to activate strict mode
  • Linting Tools - tools designed to highlight any sloppy programming practices or syntax errors; Complain if certain style conventions are not followed such as proper indentation
    • JS Lint, JS Hint, ES Lint
    • Added as a text editor plug in
    • Use an online linking tool
    • Install using NPM and run as part of your worklow
  • Some browsers support certain features others will not; do not relay on users having the most up to date browsers
    • Modernizer and Can I Use? are two useful resources for checking which features are currently supported in different browsers
  • Debugging - The process of finding out where bugs occur in the code and dealing with them
    • alert() - method to show a dialog at certain points in the code; stops a program from running until OK is clicked; puts break points in the code that let us check the values of variables a that point to see if they're what we expect them to be
    • Most modern editors have a console object that provides a number of methods for logging information and debugging
      • console.log = logs the value of variables at different stages of the program
      • console.trace() = logs an interactive stack trace in the console; shows the functions that were called in the lead up to an exception occurring while the code is running
    • breakpoints - pauses the code at a certain point; allowing you to see the values of all the variables at that point and make modifications as necessary
      • Add the debugger command in to the function to create a breakpoint in the code
      • Must be removed before finalizing your program
  • Error Objects - created by the host environment when an exception occurs; created in the code using a constructor function
    • EvalError - not used in the current EXMAScrpit specification and only retained for backwards compatibility
    • RangeError - thrown when a number is outside an allowable range of values
    • ReferenceError - when a reference is named to an item that doesn't exist
    • SyntaxError - thrown when there's in the code's syntax
    • TypeError - thrown when there's an error in the type of value used; a string is used when a number is expected
    • URIError - when there's a problem encoding or decoding the URI
    • InternalError - Non-Standard error that is thrown when an error occurs in the JavaScript engines
  • Common Errors
    • Name property returns the name of the error constructor function used as a string
    • Message property returns a description of the error and should be provided as on argument to the Error construction function
    • Stack property will remain a stack trace for that error
  • throw - used to through your own exceptions
  • try block - runs the code inside the block as normal but if an exception occurs it will pass the error object that is thrown onto a catch block
  • catch block - will only run if an exception is thrown inside it
  • finally block - added after a catch block; always executed after the try or catch blocks, regardless of whether an exception occurred or not
  • Testing is an important step in the programming process; can be a piece of code that tests different variables
  • Test Driven Development - process of writing tests before an actual code
    • 1. Write Tests
    • 2. Write Code To Pass The Test
    • Refactor The Code
    • Test Refactored Code
    • Write More Tests For New Features
  • You can write your own test but the process cab be very laborious
  • Jest - testing framework created by Facebook
    • To activate enter the command npm install -g jest into the terminal
    • Should install globally
