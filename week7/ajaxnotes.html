Ajax Notes

  • Ajax - A technique that allows web pages to communicate asychronously with a server and it dynamically updates web pages without reloading; Allowing data to be sent and received in the background as well as portions of a page to be updated in response to user events while the rest of the program continues to run
  • Same Origin Policy - The browser blocks all requests from a domain that is different from the page making the request unless Cross-Orin resource sharing are used
  • Application Programming Interface (API) - collection of methods that allow external access to another program or service
  • fetch() - only mandatory argument is the URL of the resource you wish to fetch
  • Response Interface - deals with the object that's returned when the promise is fulfilled; has an ok property that checks to see if the response is successful; 200 = response was successful; 201 = resource was created; 204 = request is successful but no content was returned
  • redirect() - used to redirect to another URL
  • text() - takes a stream of text from the response, reads it to completion and returns a promise that resolves to a USVString object that can be treated as a string in JavaScript
  • blob() - used to read a file of raw data, such as an image or spreadsheet, once it has read the entire file it returns a promise that resolves with a blob object
  • json() - method used to deal with transforming a stream of JSON data into a promise that resolves to a JavaScript object
  • HTTP Methods - Tells the server what type of request is being made, which determines how the server will deal with the request
    • Get - requirements to retrieve resources
    • Post - requirements usually used to create a resource but can actually perform any task
    • Put - requests to upsert or insert a resource or update it entirely
    • Patch - requests to make partial updates to a resource
    • Delete - requests to delete a resource
  • Request Interface - created used the request() constructor and include many properties
    • URL - the URL of the requested resource; required
    • method - a string that specifies which HTTP method should be used for the request
    • headers - a headers object that provides details of the requests headers; provide addition information such as file type of the resource, cookie information, authentication information, and when the resource was last modified
      • has() - used to check if the headers object contains the header provided as an argument
      • get() - returns the value of the header provided as an argument
      • set() - used to set a value of an already existing header or create a new header with the values provided as an argument if it does not already exist
      • append() - adds a new header to the headers object
      • delete() - removes the header provided as an argument
      • keys(), values(), entries() - iterators that can be used to iterate over the headers key, values, or entries (key value pairs)
    • mode - all you to specify if CORS (default) is used or not
    • cache - allows you to specify how the request will use the browser's cache; force it to request a resource and update the cache with the result or force it to only look in the cache for the resource
    • credentials - lets you specify if cookies should be allowed with the request
    • redirect - specifies what to do if the response returns a redirect
      • Follow - the redirect is followed
      • Error - an error is thrown
      • Manual - the user has to click on a link to follow the redirect
