  2. Reading Notes JavaScript Novice to Ninja: Chapter 12
    • Object Oriented Programming - style of programming that involves separating the code into objects that have properties and methods
    • Encapsulation - the inner workings are kept hidden inside the object and only the essential functions are exposed to the end user
    • Polymorphism - the same process ca be used for different objects; various objects can share the same methods but also have the ability to override shared methods with more specific implementation
    • Inheritance - taking the features of one object than adding new features; takes an object that currently exists and inherit all of its properties and methods, we can use the new properties and methods
    • This - keyword used to represent the object that will be returned by the construction function
    • Static Methods - add the static keyword in the class declaration; called by the class directly rather than by instances of the class; not available to instances of the class
    • Prototypal Inheritance - every class has a prototype property that is shared by every instance of the class; any properties or methods of a class's prototype can be accessed by every object instantiated by the class
    • hasOwnProperty() = used to check if a method is its own property or is inherited from the prototype
    • If a new property or method is added to the prototype any instances of its class will inherit the new properties and methods, even if the instance has already been created
    • It is not possible to overwrite the prototype by assigning it to a new object literal if class declarations are used
    • Object instance can overwrite any properties or methods inherited from its prototype by simply assigning a new value to them
    • Public Methods - can be queried directly and changed by assignment; this is the default property
    • Private Methods - impossible to access outside the scope; getter and setter methods are used to form a closure over this variable and provide controlled access to the property inside
    • Object.getPrototypeOf() = shows us the prototype of the prototype is an apparently empty object lateral
    • propertyIsEnumerable() = checks to see if a property is enumerable
      • All built in methods should be non-enumerable
      • All user defined methods should be enumerable
    • Extends = allows a class to inherit from another class
    • Monkey Patching - adding additional methods to the prototype of JavaScript's built in objects; should be avoided unless absolutely necessary
    • All object properties have the following attributes
      • Value - values of the property and is undefined by default
      • Writable - boolean value shows whether a property can be changed or not; false by default
      • Enumerable - boolean value shows whether a property will shows up when the object is displayed in a for in loop; false by default
      • Configurable - boolean value shows whether you can delete a property or change any of its attributes; false by default
    • Object() = creates a new object based on another object
    • Mixin - Way of adding properties and methods of some object to another object without using inheritance
    • Factory Function - a function that can be used to return an object
    • If a method returns this its method can be chained together to form a sequence of method calls that acre called one after another
    • That can sometimes go out of scope when nested in other functions
      • Set the variable that to equal this before the nested function; later refers to the nested function and not the this
      • bind() = method for all functions and is used to set the value of this in the function
      • For Of = syntax for arrays and does not require a nested function to be used so this remains bound to the object
      • Using Arrow Functions - Arrow functions do not have their own context, so this remains bound to the original object making the function call
      • Methods can be borrowed from objects without having to inherit all their properties and methods by making a reference to the function that you want to borrow
      • call() = takes the object that the function is to be applied to as its first argument, then the usual arguments come afterwards
  3. Reading Notes JavaScript Novice to Ninja: Chapter 15
    • Module - self contained pieces of code that provided functions and methods that can then be used in other files any by other methods; helps keep code organized in separate reusable files
    • ES6 Modules
      • All code in modules is always in strict mode without the need for 'use strict' and there is no way to opt out of this
      • A module has its own global scope, so any variables created in the top level of a module can only be accessed within the module
      • The value of this in the top level of a module is undefined, rather than the global object
      • You cannot use HTML style comments in modules
    • Module files use the keyword export to specify any values of functions that are to be made available in the module
    • Not everything in a module needs to be used
    • Import is used to add the modalized functions into the main JavaScript file
    • * = wild card symbol used to indicate everything in the module file should be imported
    • Default Exports - a single variable, function, or class in a module that can be imported without having to be explicitly named; can only be used once
